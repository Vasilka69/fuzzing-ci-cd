# Инфраструктура для вайбкодинга CI/CD executor-системы

Дата подготовки: 2026-05-29.

Этот пакет содержит Markdown-инфраструктуру для agent-friendly разработки дипломного проекта: `AGENTS.md`, верхнеуровневые `PRD.md` и `TASKS.md`, отдельные PRD/TASKS по каждому исполняющему микросервису, service-scoped инструкции для агентов, чеклисты, шаблоны промптов и архитектурные решения.

## Что важно по scope

Реализуются только исполняющие микросервисы и общие модули, необходимые для них:

- `vcs-service`
- `storage-service`
- `build-service`
- `fuzzing-service`
- `deploy-service`
- `script-service`
- `cicd-common`, `cicd-executor-core`, `cicd-test-support`

`ui` и `master-service` не реализуются в этом workstream. Для них описаны только контрактные границы: Kafka topics, message schemas, OpenSearch documents, artifact URI и ожидания по статусам.

## Рекомендуемая структура использования

1. Скопировать содержимое этого пакета в корень будущего репозитория.
2. При старте разработки сначала дать агенту прочитать `AGENTS.md`, затем только релевантные PRD/TASKS конкретного сервиса.
3. Для работы внутри конкретного модуля использовать `services/<service>/AGENTS.md`, чтобы не перегружать контекст нерелевантными требованиями.
4. Задачи выполнять по уровням из `TASKS.md`: сначала MVP, затем дипломно-достаточный функционал, затем hardening. Опциональные production-задачи вынесены в конец.

## Содержимое

| Файл/папка | Назначение |
| --- | --- |
| `AGENTS.md` | Главные инструкции для AI coding agents. |
| `PRD.md` | Верхнеуровневый product requirements document. |
| `TASKS.md` | Верхнеуровневый backlog по уровням. |
| `docs/prd/` | PRD по отдельным исполняющим микросервисам. |
| `docs/tasks/` | TASKS по отдельным исполняющим микросервисам. |
| `services/*/AGENTS.md` | Локальные инструкции, которые сужают контекст до одного модуля. |
| `docs/context/` | Контекст проекта, БД, структуры репозитория, AI workflow. |
| `docs/architecture/` | ADR по Maven-модулям, контрактам executor-ов и fuzzing kernel. |
| `docs/checklists/` | Чеклисты ревью, безопасности и Definition of Done. |
| `docs/templates/` | Шаблоны задач и планов перед кодом. |
| `docs/prompts/` | Библиотека промптов для агентов. |
| `docs/k8s/` | Требования к Docker/Kubernetes-развертыванию. |
| `.github/pull_request_template.md` | PR-шаблон под agentic workflow. |

## Базовый принцип

Агент не должен сразу писать код. Сначала он объясняет, как понял задачу, предлагает один основной и 1-2 альтернативных подхода, фиксирует риски, план проверки и ожидаемый diff. Код пишется только после явного согласования подхода пользователем.
