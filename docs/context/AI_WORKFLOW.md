# AI workflow для разработки

Этот документ описывает рабочий процесс AI-агента при разработке.
Полные обязательные правила находятся в корневом `AGENTS.md`.
Шаблон ответа перед реализацией находится в `docs/templates/PLAN_RESPONSE_TEMPLATE.md`.
Canonical Definition of Done находится в `docs/checklists/DEFINITION_OF_DONE.md`.

## Перед кодом

Агент обязан сначала подготовить план по шаблону из `docs/templates/PLAN_RESPONSE_TEMPLATE.md`.

Код писать только после подтверждения подхода пользователем.

## Во время реализации

- Делать минимальные итерации.
- Не менять нерелевантные сервисы.
- Не добавлять зависимости без согласования.
- Сохранять контрактную совместимость.
- Писать тесты вместе с кодом.
- Сложную логику документировать Javadoc/комментариями.

## После реализации

Итоговый ответ агента:

```markdown
## Что изменено

## Почему так

## Как проверено

## Риски и ограничения

## Что осталось
```

## Когда нужен новый ADR

- Меняется внешний Kafka/OpenSearch JSON contract.
- Меняется Maven/module layout.
- Добавляется production dependency.
- Ослабляется sandbox/security policy.
- Меняется способ интеграции fuzzing kernel.
- Добавляется прямой доступ executor-а к какой-либо внешней системе, не предусмотренной PRD.
