# Definition of Done

Этот файл является единственным canonical источником Definition of Done для проекта.
`AGENTS.md` может ссылаться на этот файл и содержать краткий gate для агента, но не должен дублировать полный чек-лист.

При изменении критериев готовности обновляй этот файл.

- [ ] Задача соответствует PRD/TASKS конкретного сервиса.
- [ ] Перед кодом был согласован подход.
- [ ] Нет расширения scope без подтверждения.
- [ ] Добавлены/обновлены тесты.
- [ ] Пройдены релевантные команды Maven.
- [ ] Публичные JSON-контракты совместимы.
- [ ] Нет секретов в коде, логах, тестовых данных, artifacts.
- [ ] Ошибки классифицируются через единый `error.type`.
- [ ] Workspace очищается согласно policy.
- [ ] Docker/Kubernetes обновлены, если менялся runtime.
- [ ] Сложная логика документирована Javadoc/комментарием.
- [ ] Временные, заглушечные или неполные места явно помечены `TODO` с причиной и ожидаемой дальнейшей доработкой.
- [ ] Итоговый ответ содержит проверки и оставшиеся риски.
- [ ] Итоговый ответ содержит: что изменено, почему, как проверено, как прошла работа, риски;
- [ ] В случае, если задача взята из списка из `TASKS.md` и считается полностью выполненной, она помечается, как готовая;
- [ ] Проверено, нужно ли обновить документацию и `docs/MANIFEST.json`; документация обновлена либо явно указано, почему обновление не требуется.
