# Review checklist

## Функциональность

- [ ] Решена исходная задача.
- [ ] Не добавлено лишнее поведение.
- [ ] Обработаны negative/error cases.
- [ ] Повторная доставка job не ломает результат.

## Архитектура

- [ ] Изменение вписывается в Maven module boundaries.
- [ ] Executor не зависит от master-service/ui.
- [ ] Общие DTO не дублируются по сервисам.
- [ ] Большие данные передаются через URI, не через Kafka.

## Контракты

- [ ] `schemaVersion` корректен.
- [ ] JSON поля camelCase.
- [ ] `jobExecutionId` везде присутствует.
- [ ] `JOB_FINISHED.logs` пустой/null.
- [ ] `JOB_LOG` содержит лог-фрагменты.

## Тесты

- [ ] Есть unit tests для новой логики.
- [ ] Есть integration/contract tests там, где меняется интеграция.
- [ ] Тесты проверяют поведение, а не внутренности реализации.
- [ ] Проверки не отключены.

## Документация

- [ ] Обновлены PRD/TASKS/README, если изменился scope.
- [ ] Javadoc есть для сложной логики.
- [ ] Known limitations записаны честно.
